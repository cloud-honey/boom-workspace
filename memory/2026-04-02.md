   - 기존 DuckDuckGo 검색은 이미 구현됨
   - 검색 트리거 키워드 확장
     * 의문사 자동 감지: 누구, 언제, 어디, 무엇, 어떤, 왜, 몇
     * 정보성 질문: 정보, 정의, 뜻, 의미, 요청, 취업, 채용, 공고

   - 사용처 명시:
     * 사실 확인 필요 + 검색 결과 없음

     * 개인 의견 요청 ('뭐가 좋아?') → '모르겠습니다'
     * 최신 정보 (훈련 데이터 부족)
   - 예시로 명확히: 
     * 나쁜 답: "모르겠습니다." (그냥 회피)
     * 좋은 답: "어제 소식을 찾을 수 없습니다." (명확한 이유)

     * deque로 자동 오래된 항목 삭제
   - 메시지 전송 시 히스토리 포함
     * query_mlx() 함수에 user_id 파라미터 추가
     * 대화 흐름 자연스럽게 이어짐
   - /start로 히스토리 초기화 가능
   - 메시지 형식: {"role": "user"/"assistant", "content": "..."}
