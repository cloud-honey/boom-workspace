# Stock Insight — 기획서 v0.1

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Stock Insight (가칭) |
| 목적 | 보유 종목 관리 + 기술적 분석 → 매수/매도/보유 신호 제공 |
| 형태 | React 대시보드 (PC/모바일 반응형) |

## 2. 핵심 기능

### 2-1. 증권사 계좌 연동 (OCR 이미지 업로드)

**문제:** 증권사 5개 직접 입력 힘듦

**해결:** 캡처 이미지 업로드 → AI가 데이터 추출 → 자동 등록

```
[사용자 업로드 흐름]
계좌 캡처.png → AI OCR → 종목명 + 수량 + 평균가 추출 → 확인 → 등록 완료
```

- **OCR 엔진:** Gemini 2.0 Flash (이미지 분석)
- **추출 필드:** 증권사명, 계좌번호, 종목명, 보유수량, 평균매입가
- **추출 후:** 사용자가 내용 확인/수정 후 확정

**지원 예상 증권사:**
삼성, KB국민, 대신, 미래에셋, 한국포国际金融 등 → 이미지 학습으로 추가 가능

### 2-3. 기술적 분석 (매매 신호)

| 지표 | 설명 | 신호 기준 |
|------|------|----------|
| **이동평균선(MA)** | MA5 > MA20 → 골든크로스 | 매수/매도 신호 |
| **RSI** | 70 이상 과열, 30 이하 과매도 | 매도/매수 신호 |
| **MACD** | MACD > 시그널 → 상승추세 | 매수/매도 신호 |
| **볼린저밴드** | 하단 근처 → 반등 가능성 | 매수 신호 |
| **종합 신호** | 다수 지표 합산 | 매수/보유/매도 |

**종합 신호 산출 로직:**
- 매수 신호 3개 이상 → 매수 추천
- 매도 신호 3개 이상 → 매도 추천
- 그 외 → 보유

### 2-4. 증권사별 계좌 관리

- 증권사별 계좌 카드 표시
- 계좌별 수익률, 보유종목 목록
- 전체 포트폴리오 vs 개별 계좌 조회 토글

## 5. OCR 이미지 업로드 상세

**업로드 절차:**

1. 사용자가 계좌 캡처 이미지 업로드 (.jpg/.png)
2. React → Flask로 이미지 전송
3. Flask → Gemini API로 분석 요청
4. Gemini 응답 파싱 → JSON 추출:
   ```json
   {
     "broker": "삼성증권",
     "account_no": "123-45-67890",
     "stocks": [
       {"name": "삼성전자", "quantity": 50, "price": 72000},
       {"name": "SK하이닉스", "quantity": 20, "price": 180000}
     ]
   }
   ```
5. React에서 확인 UI 표시 → 수정 → 저장
6. DB 저장 → 대시보드 반영

**학습 필요:** 5개 증권사 수동 매핑 (증권사명 → 이미지 패턴)
