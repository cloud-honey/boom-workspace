# 붐엘 지속적 최적화 아키텍처 설계

## 마스터 요구사항 (11:22 KST)
1. **대화 패턴 학습**: 마스터와의 대화 방식 자동 학습
2. **지속적 최적화**: 매 대화마다 점점 더 똑똑해짐
3. **개인화**: 마스터의 선호도/습관을 반영한 응답
4. **장기 메모리**: 과거 대화에서 배운 것을 기억
5. **아키텍처 통합**: 모든 요구사항을 하나의 시스템으로

---

## 📋 현재 상태 분석

### 기존 기능 (v1)
- ✅ 웹 검색 (DuckDuckGo)
- ✅ 대화 히스토리 (최근 10개 메시지, 세션 범위)
- ✅ 시스템 프롬프트 (붐의 성격)
- ❌ 장기 메모리 (세션 끝나면 사라짐)
- ❌ 대화 패턴 분석
- ❌ 개인화 학습
- ❌ 성능 측정/피드백

---

## 🎯 설계 목표

### 1단계: 장기 메모리 기반
- 마스터와의 모든 대화 저장 (DB)
- 세션 간 히스토리 유지
- 과거 대화 컨텍스트 활용

### 2단계: 대화 패턴 분석
- 마스터의 응답 방식 분석
  * 선호하는 답변 길이
  * 선호하는 말투 (반말/존댓말, 기술용어 포함 정도)
  * 관심 분야 (경제/기술/일상)
  * 반복되는 질문 패턴

### 3단계: 동적 프롬프트 조정
- 마스터의 특성에 맞게 시스템 프롬프트 자동 생성
- 예: 마스터가 기술 질문을 많이 하면 → 기술 설명 강화
- 예: 마스터가 짧은 답변을 좋아하면 → max_tokens 더 줄임

### 4단계: 피드백 루프
- 마스터의 반응을 통해 개선도 측정
- "이건 좋아" / "이건 싫어" 같은 피드백 수집
- 명시적 평가 버튼 추가

## 🔄 실행 흐름 (Detailed)

### A. 대화 입력 시
```
1. 마스터 메시지 수신
2. 사용자 프로필 로드
   - 선호도: short answers, casual, finance focused
3. 히스토리 로드
   - 세션: 최근 10개 (현재 대화)
   - 장기: 최근 30일 유사 질문 TOP 5
4. 컨텍스트 검색
   - "금융" 관련 질문이면 지난 금융 대화 참고
   - 유사도 기반 검색 (임베딩)
5. 동적 프롬프트 생성
   ```
   기본 성격 (붐의 SOUL.md)
   + "이 사용자는 금융에 관심이 많고 짧은 답변을 선호합니다"
   + "최근 코스피 관련 질문 3회, 투자 조언 2회"
   + "사용자의 평가: 전반적으로 만족 (84%)"
   ```
6. MLX 모델 호출
7. 응답 기록 + 메트릭 저장
```

### B. 피드백 수집 시
```
1. 마스터가 "좋아요" / "싫어요" 반응
2. 대화 레코드에 피드백 기록
3. 학습 데이터 생성
   - 이 응답이 왜 좋았는지 분석
   - 패턴 추출 (예: "짧은 기술 설명" 좋음)
4. 사용자 프로필 업데이트
5. 신뢰도 증가 (learning_curve)
```

### C. 주기적 최적화 (예: 매일 자정)
```
1. 마스터의 지난 24시간 대화 분석
2. 새로운 패턴 감지
3. 프로필 갱신
4. 추천 개선사항 제시 (선택)
```

## 🛠️ 기술 스택

### 저장소 (Storage)
- **SQLite** (로컬, 간단함)
  또는
- **PostgreSQL** (확장성, 나중에 다중 사용자)

### 검색 (Retrieval)
- **임베딩**: OpenAI text-embedding-3-small
  또는
- **로컬 임베딩**: sentence-transformers (Ollama 내부)
- **벡터 DB**: FAISS (로컬) 또는 pgvector (PG)

### 분석 (Analysis)
- **통계**: numpy/pandas
- **패턴 추출**: 규칙 기반 또는 간단한 ML

### API
- **기존**: FastAPI (MLX 서버)
- **신규**: 대화 기록/프로필 관리 API
