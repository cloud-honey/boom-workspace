## 6. 현재까지 가장 중요한 기술적 발견

### 1) 모델이 죽은 게 아님
직접 실행에서는 응답이 나온다.
즉, “붐4 모델이 아예 작동 안 한다”는 문제는 아님.

### 2) 문제는 서브에이전트 실행 구조 쪽
같은 모델이 직접 실행에서는 응답하고, 서브에이전트 경로에서는 timeout만 난다.
따라서 원인은 아래 계층일 가능성이 높다.
- 샌드박스 실행 경로
- OpenClaw subagent tool bridge
- boom4 agent runtime configuration
- local model + sandbox I/O 연동 방식

### 3) 권한/메모리 문제 가능성은 낮아짐
- 메모리 로드 확인 후에도 실패
- 권한 완화 후에도 실패
- 모델 교체 후에도 동일 패턴

즉, 단순 권한/메모리 부족 이슈로 보기 어렵다.

4. 작업은 큰 구현보다 **작은 수정 단위**로 쪼개서 사용할 것

2. 다른 로컬/서브에이전트(예: 붐3)와 동일 초소형 테스트 비교
