# 데일리 리포트 생성 규칙 수정안 — 2026-03-30

## 문제 요약
현재 리포트는 검색 결과 snippet을 그대로 조합하는 방식이라 다음 문제가 반복됨.
- 메인/섹션 페이지가 기사처럼 섞임
- 오래된 기사와 최신 기사가 섞임
- 본문 확인 없이 설명문만 보고 문장 생성
- 칼럼/해설/유튜브/포털 모음 페이지가 뉴스로 채택됨
- 같은 문구가 여러 날 반복 재사용됨

## 오늘 확인된 오염 패턴
### 아침 리포트
- 오래된 금리 인상 기사 재사용
- 근거 불명확한 OpenAI/Gemini 일반론 포함
- 해설형 문장과 전망형 문장이 기사처럼 포함

### 오후 리포트
- 경제 섹션 메인 페이지/포털 묶음 페이지 기반 문장 생성
- 오늘 기사와 무관한 금리 기사 재사용
- 반도체/AI 섹션에 일반론·반복문장 혼입

## 바로 적용할 생성 규칙
1. 검색 결과에서 아래 URL/출처는 즉시 제외
- 메인 페이지, 섹션 페이지, 포털 허브, 검색 허브
- 예: /economy, /money, /news, /world, /it, 홈 URL
- Daum/Naver 뉴스 모음 페이지, YouTube, 블로그, 포럼 제외

2. 기사 채택 조건
- 개별 기사 URL만 허용
- published 또는 기사 날짜가 현재 시점 기준 48시간 이내여야 함
- 날짜 확인 불가 기사 제외
- 같은 사실을 반복하는 중복 기사 제거

3. 검색어 방식 변경
- 기존: "오늘 한국 경제 뉴스"
- 변경: 구체 쿼리 3~5개로 분리
  - 경제: "site:yna.co.kr OR site:newsis.com OR site:hankyung.com OR site:mk.co.kr 한국 경제 2026-03-30"
  - AI: "site:openai.com OR site:anthropic.com OR site:googleblog.com OR site:techcrunch.com AI 2026-03-30"
  - 반도체: "site:yna.co.kr OR site:newsis.com OR site:hankyung.com OR site:mk.co.kr 삼성 SK하이닉스 TSMC 엔비디아 2026-03-30"

4. snippet 단독 사용 금지
- 검색 결과 제목/설명만으로 리포트 문장 작성 금지
- 채택 후보는 반드시 web_fetch 또는 기사 본문 확인 1회 후 요약
- 본문 확인이 실패하면 해당 기사 제외

5. 섹션별 최대 건수 축소
- 경제 2건
- AI 2건
- 반도체 2건
- 많아도 3건 초과 금지
- 불확실한 내용을 채우려고 억지로 추가하지 말 것

6. 표현 규칙
- 전망/칼럼/분석/가능성/예상 문장은 사실 기사와 구분해서만 사용
- 확인되지 않은 수치·상장 계획·최초 탑재 같은 표현은 보수적으로 작성
- "알려졌다", "전망" 같은 표현도 본문 근거가 있을 때만 사용

7. 자가 검증 단계 추가
리포트 작성 전 체크리스트:
- 모든 뉴스 항목이 개별 기사 URL인가?
- 날짜가 48시간 이내인가?
- 본문 확인을 거쳤는가?
- 중복/오래된 문장이 섞이지 않았는가?
- 오늘 시점의 핵심 이슈와 맞는가?

## 테스트 기준
테스트 생성 후 다음을 확인
- 금리 인상 같은 오래된 이슈가 다시 나오지 않는지
- 기사 출처 없이 일반론이 섞이지 않는지
- 아침/오후 리포트 문장이 서로 복붙처럼 반복되지 않는지
- 실제 기사 제목과 요약 방향이 대체로 일치하는지
